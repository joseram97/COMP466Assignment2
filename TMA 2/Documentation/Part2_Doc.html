Assignment 2 - Part 2 Documentation

Description and Requirements

This part of the assignment involves creating an online web course site that will enable learners to sign up and take some courses created by subject matter experts (SMEs). This project is to use the knowledge gained from using php and MySQL to create a seemless experience. The requirements will be listed below, based on their sections

Here are the requirements

  • Create a multi-paged online learning course website to help users learn about various topics the Subject Matter Experts (SMEs) create
  • Utilize a simple login/log-out system to keep track of the "learning" users and the SMEs
  • Utilize an Education Mark-up Language (EML) for SMEs to use and create their courses for the site
  • The EML should be able to
    • Allow SME to create units
    • Allow the SME to create lessons within the units
    • Allow the SME to add images to lessons to enhance learning
    • For every unit created, a quiz is also to be setup with as many questions as they like
    • Be setup in an easy to read and use form
  • The website must be able to parse the EML and display it as html content for the user to view the courses, units, lessons and quizzes. This is done with the creation of a parser
  • The website must be able to
    • Allows users to login and signup
    • Allows "learners" to enroll/un-enroll courses available on the website
    • Allows SMEs to upload an EML based file (along with the option to upload images with the EML file as well)
    • Allow SMEs to view the courses that they uploaded to ensure the quality of the course
    • Allow SMEs to update their courses with a new EML file (and newer images)
    • SMEs can delete any course that they've created
    • Both Learners and SMEs must be able to view Quizzes on a seperate web page and take the test, showing results
  • The website must be presented in a professional manner, with a simple to understand user-interface

Website Layout

The entire course will be structured with 3 pages

  • Home Page/Course Management Page
    • This page will be the "main home page" for the entire course, both for logged in users and non-logged in users. The Home Page part will showcase some of the available courses from the website. It will also include the Log in and Sign up buttons for a user to either create a new account, or log in with their existing credentials. Once a user logs in or signs up, the page will then change to the Course Management page. This new page will be different for either Learners and Subject Matter Experts. For Learners they will be shown 2 lists. One list will showcase all of the courses the user is enrolled in, and another list showcasing all of the available courses given by the website. The Learner will be able to enroll and un-enroll within this page. For the SME within the same page, they will only be shown 1 list, showcasing all of the courses that they have created and uploaded to website. For both types of users, this page will allow users to click and select a course to view in the Course Viewer page and see the quizzes in the Course Quiz viewer page.
  • Course Viewer
    • This page will show the same course contents for the Learner and SME. For the Course Viewer page, it will show all of the course contents, including the units within the courses, their lessons, and any image media. These contents will be grabbed with the database server. Additionally, another page called the Course Quiz page will showcase all of the available quizzes for each of the units within the course.
  • Course Quiz Viewer
    • This page will show the same course contents for the Learner and SME. The page called the Course Quiz page will showcase all of the available quizzes for each of the units within the course. The contents will be grabbed by the database server.
NOTE: all of the pages will also include a Sign out button to sign out and redirect to the home page at any time.

EML Design

The Educational Markup Language should be designed to be easy to understand and easy to create courses. The following tags were set up with corresponding attributes.

  • course - the main wrapper to hold all other course content
    • Attribute: name - the main title of the course
    • courseDescription - a brief description of the entire course
    • units - holds all of the individual units for the course
      • unit - the individual unit
        • Attributes: number - the unit number, title - the unit title
        • unitDescription - a brief description of what the unit will include
        • lesson - will contain all of the lesson contents and images
          • Attributes: number - the lesson number, title - the lesson title
          • lessonContent - a paragraph of text which may include images
            • Attributes: number - the content number, this will help with ordering the paragraph, title - the lesson content title (optional) for more detail
              • content - the main text for the paragraph. May require !CDATA wrapper (you can see a sample later)
              • images - the tag to hold all of the images that may be added right after the paragraph
                • image - the main tag for "saving" the image
                  • Attributes: number - the order number of the image to show beneath the paragraph, fileName - the name of the file that will be uploaded to then link to the tag and then be saved on the server
    • quizzes - holds all of the individual quizzes for the course. Must be equal to the number of units for the course
      • quiz - the individual quiz element that will hold the question and answers
        • Attribute: unit - the corresponding unit number that the quiz is linked to
        • question - the individual question element that will hold the main question text, and the answers
          • Attribute: number - the order number for the question
            • questionText - the text for the question
            • answer1 - the first answer
            • answer2 - the second answer
            • answer3 (optional) - the third answer
            • answer4 (optional) - the fourth answer
            • correctAnswer - the number of the correct answer, between 1-4

Database Design

The database for the website will consist of 8 tables. Each of these tables have a specific purpose of which data to store. See below a description of each of the tables and their relations:

  • Users

    • This table will contain all of the user information

      Fields

      • username - the username that the user wants to login as
      • password - the password for the user to login with
      • fullname - the full name of the user

  • Courses

    • This table will contain the courseid and its name, and description, and will be the "base" of keeping track of the courses

      Fields

      • courseid - the courseid for the course that is created
      • courseName - the name/title of the course
      • courseShortDescription - a brief description of what the course is about
      • userCreator - the username of the SME who created the course related to username from Users
      • createdDate - the date of when the course was created

  • UserCourseAssignments

    • This table will contain the user assignments (learners) enrolled with the courses that they selected

      Fields

      • username - this is the username of the user enrolled must exist in Users
      • courseid - this is the courseid of the course the user is enrolled in must exist in UserCourseAssignments

  • Units

    • This table will contain the units for a given course and the unit information

      Fields

      • unitNumber - the order number of the unit within the course
      • courseid - the courseid that the unit is associated with must exist in table Courses
      • unitTitle - the title name of the unit
      • unitDescription - a brief description of the unit

  • Lessons

    • This table contains the lessons and lesson information for a given unit within a course

      Fields

      • lessonNumber - the order number of the lesson within a unit
      • unitNumber - the associated unitNumber that the lesson is a part of must exist in Units
      • courseid - the associated courseid that the lesson's unit is a part of must exist in Units
      • lessonTitle - the title name of the lesson

  • LessonContents

    • This table contains the actual content information of the associated lesson

      Fields

      • lessonNumber - the order number of the lesson within a unit must exist in Lessons
      • unitNumber - the associated unitNumber that the lesson is a part of must exist in Lessons
      • courseid - the associated courseid that the lesson's unit is a part of must exist in Lessons
      • contentNumber - the order number of the content paragraph within the lesson
      • title - the title name (optional) of the lesson content
      • content - the main text content of the paragraph

  • Images

    • This table contains the image url that is associated with the lesson contents

      Fields

      • imageNumber - the order number of the image within the lesson contents
      • lessonNumber - the associated lessonNumber must exist in table LessonContents
      • unitNumber - the associated unit number must exist in table LessonContents
      • courseid - the associated course id must exist in table LessonContents
      • contentNumber - the associated lesson content number must exist in table LessonContents
      • imageUrl - the image url of where the image is located in the site server

  • Quizzes

    • This table contains all of the quiz information that is associated with the course (must be the same amount of units)

      Fields

      • questionNumber - the order number of the question within the unit
      • unitNumber - the associated unit number that the questions are a part of must exist in table Units
      • courseid - the courseid that the unit question is a part of must exist in table Units
      • questionDescription - the main question text for the question
      • answer1 - the text of the first answer
      • answer2 - the text of the second answer
      • answer3 (optional) - the text of the third answer
      • answer4 (optional) - the text of the fourth answer
      • correctAnswer - the number of the answer that is correct (e.g. 1, 3, 2)

Site Data Flow

The data flow within the site is comprised of using php server and enabling cookies for a smooth experience. The cookie fields were designed as follows:

  • loggedIn - is a boolean and is true when a user signs up or logs in, false if logged out
  • username - the string username of the user that is logged in
  • fullname - the full name string of the user
  • usertype - can be either "Learner" or "SME". This is set when a user is logged in/signed up
  • courseSelected - is a boolean and is true when a course is selected from the "Course Management" page
  • courseid - the course id of the selected course
  • unitSelected - is as boolean and is true when a user selects a unit from the "Course Viewer" page
  • unitNum - the unit number selected from the course on the "Course Viewer" page
  • quizSelected - is a boolean and is true when a user selects a quiz from the "Course Quizzes" page
  • quizNum - ths quiz number selected from the quiz on the "Course Quizzes" page

The flow of data will be described as follows:
  • Logging in as a user
    • User enters username and password *click Log In -> php query MySQL database (users table) and verify that the username exists, and password matches *return true or false -> Save username, fullname, and usertype to the cookies *redirect user to homepage (a.k.a the Course Management page now
  • Signing up as a user
    • User enters username, usertype, password *click Sign Up -> php query insert to MySQL database (ensuring username is not already taken) *return success or failure -> Save username, fullname, and usertype to the cookies *redirect user to homepage (a.k.a the Course Management page now
  • SME creating a course
    • SME uploads EML file (and optionally images) to server *click Upload -> parse through the EML file *get course info -> parse through course info *get unit info -> parse through unit info *get lessoninfo -> parse through lesson info *get images and contents (if any) -> save images onto the server -> parse through quiz information *insert all contents into database insert EML contents into the MySQL database (course, units, lessons, lesson Contents, images, quizzes) *complete insertion -> show SME new course within course list
  • SME deletes course
    • SME selects a course to delete *click delete -> delete images from the server -> sends the course id and php will remove the course from the Courses table *MySQL cascade delete -> show the new course list to the SME user that the course is no longer there
  • SME edits course
    • SME selects a course to edit *click edit -> user uploads the edited EML file (along with any images) -> delete images from the server -> sends the course id and php will remove the course from the Courses table *MySQL cascade delete -> upload the course EML and images -> show the updated date and time of the course on the SME course list
  • Learner enrolls in course
    • Learner selects a course from a list of available courses on the website *click add course button -> php then sends insert request to add, in UserCourseAssignments table, a connection between the username and the course id *executes query -> the user is then shown the course is now added in the enrolled course list
  • Learner un-enrolls in course
    • Learner selects a course to remove from their enrolled courses list *click remove button -> php removes the username, and courseid association within the UserCourseAssignments table *executes query -> The user now see's the course is not in their enrolled courses and is back in the available courses list
  • User viewing course
    • User selects course from the Course Management page OR User opens the Course viewing page -> in cookies, courseSelected is set to True, courseid is set to the course id selected *sets cookie and redirects to Course Viewer page -> User selects a unit *click on unit -> in cookies, unitSelected is set to true, unitNum is set to the selected unitNum -> php querys with course and unit cookie information and grabs the course information about unit, lessons, and images, and displays it as html *return course contents -> the user now can view the unit selected and read through the material
  • User viewing Quizzes
    • User opens the Course Quiz viewing page -> User selects a quiz *click on quiz -> for cookies, set quizSelected to true, and quizNum is set to the quiz number selected *save cookies -> php queries and uses quiz, unit, and courseid cookie information and grabs the quiz information based on the courseid selected from the course management page *return quiz questions -> user takes the quiz and submits the result *click submit -> javascript will verify the answers selected and calculate the number of correct answers and the percentage *returns correct answers and percentage -> user is shown the correct answers, the number of correct answers selected, and the percentage of correct answers

EML Parser

The EML Parser is relatively simple and has been partially described earlier. But here is the process

  • Subject Matter Expert (SME) uploads the EML file (along with photos)
  • PHP will read the EML.xml file and extract and go through the course, unit, lesson, lessonContent, Image, and Quiz information and insert into the MySQL database
  • Now, whenever the a user wants to view a course, after the course is selected PHP server will query the MySQL database and return the course, unit, lesson, lessonContent, image data and added them to created html objects
  • Whenever a user wants to view a quiz, after a quiz is selected PHP server will query the MySQL database to get the quiz data (also based on the course that is selected)

Implementation

The entire website was created using HTML, CSS, PHP, MySQL, and XML technologies. I have tried to make the desktop experience as consistent as possible, sharing the same styles between the course, quiz, and viewer pages. I have kept all of the files within the same folder directory in order to maintain simplicity, and I thought that have 3 different pages (1 for course management, 1 for course viewing, and 1 for quiz taking) helps organize the code a little better.

Within the part2 folder I have the following sub-folders and their purpose:

  • EML - holds some of the sample EML.xml data to be referenced by a user
  • phpLibrary - holds seperate php files holding functions that are to be used within the main .php files in the part2 folder
    • EMLFunctions - this php file contains functions for uploading and parsing out an EML file (with images) and transferring contents to a MySQL database. Also contains additional helper functions
    • queryFunctions - this php file contains SQL query helper functions to make querying a little bit easier within the main .php pages in the part2 folder
  • SME_Images - this folder on the server will hold all of the saved images for its courses
  • Test_Images - a folder to hold sample image data to be uploaded to the server again. Only have access to this in the local zip file, the test_images will not be stored in the server

When setting up the php server within the hosted server, some additional extensions had to be made in order to accomodate larger image and uploads made when creating courses. The following are the following php.ini variables that were modified:

  • post_max_size=50M - the max size that a total file uploads can be is 50 MB
  • upload_max_filesize=15M - the max size that a single file uploaded can be 15MB
  • max_file_uploads=75 - the max number of files that can be uploaded is 75

Lastly, the way the EML process is setup, within some of the tags used, CDATA is highly recommended to ensure that it is easier to write additional html elements within some contents. This is helpful for some SMEs that want to bold, italisize, create ordered or unordered lists using strong, i, ul, ol list tags. This is evident in the sample EML.xml file that is link in the design description.

User Guide

The following is the user guide explaining how to use the site as a learner and/or as an SME, what pages are available, how to view courses and finally take quizzes.

Homepage

When you first open the site, you will be greeted with the homepage of the B.O.A.T. Online Course website. From here you can view some of the courses that are available to learn from when you sign up for the website. You can see there is a link to the main TMA 2.html page near the main title of the page. You will also notice that there is a navigation bar on the top of the page. At first if you are not logged in, it should only show "home". There is both a Log In and Sign Up button near the top right of the page.

You will also notice near the bottom of the page a "list" of courses that are available within the course that a Learner may be enroll in.

To Sign Up, click the Sign up button on the top right of the page, and follow all of the prompts, inputs within to create your user profile. NOTE: One thing to keep of note, when you create the user profile, you will also need to select the type of profile you want. You have the choice between a Learner or a Subject Matter Expert (SME). This is important. If you want to be a Learner, select Learner, otherwise select SME. Some errors may pop up, and follow their suggestions to resolve the issues (e.g. username already taken, wrong password input). Once you complete the profile, you will be automatically logged in to the page.

To Log In, click the Log in button on the top right of the page. You will be prompted to enter your username and password (this is assuming that you have created a profile from the Sign up button). Once the correct username and password are entered, you will be logged into the site and be shown the Course Management page.

NOTE #1: If you are an SME, you will be able to manage courses that you have created, and be able to view courses and take "mock" quizzes

NOTE #2: To sign out anytime, click the "Sign Out" button on the top right of the page

Course Management - Learner

When you log in as a Learner, you will notice that you are directed to the Course Mangement page. From here you will be shown 2 lists. One for the courses that you are enrolled in, and another showing all of the courses that are available on the website. As learner, you can enroll in courses by clicking the + button on any of the courses. To un-enroll in courses, on the left list, simply click the "trash" remove button to not have the course. You may view the course and its units by simply clicking the area where it says Click here to select the course and you will be redirected to the Course Viewing page

Course Viewing

Within this page, you will now notice that there are multiple navigation buttons near the top of the page.

  • Course Management: will take you back to the course management page to view courses available on the site (or if you are an SME, add/remove courses that you have created)
  • Course Viewer: will open the main course page to be able to view all of the course material, including the units and lessons
  • Course Quizzes: view all of the available quizzes for each of the units

On the left of the page is the Unit Selector pane, where all of the available units are visible and to view them, simply click on the unit name that you desire. The unit will be visible within the page, and you can view and study the material.

Quizzes

Within this page, you will be able to quiz yourself on the units that were available within the course. On the left, you will see the Quiz Selector pane. To select a quiz, click on any of the quiz names (based on their unit number) and all the questions will be visible. Please ensure to select an answer for each of the questions before clicking Submit. Once you click Submit the number of correct answers will be visible, along with your percentage mark. To view any wrong answers, simply scroll through the questions and look at a question that resulted in showing a red box. There, it will tell you what the correct answer is so that you may learn and try again the next time.

Course Management - SME

As a Subject Matter Expert you will be given power to create courses that other users can sign up for! After signing up as an SME, you will be redirected to the Course Management page. From here, it will look a little bit more different than when a Learner has logged in. You will notice that there is only a single list. This list will show all of the courses that you, as an SME, have created for the website. You can create/edit/remove any of these courses

Course Creation - SME

To create a course, you have to understand the Educational Markup Language (EML) used for organizing the contents of the course. Your course may include:

  • Units
  • Lessons for each unit
  • Quizzes (1 for each unit)
  • Image media

In order to make the creation of the courses easier, the EML has been set up with easy to understand tags. To view an example of the EML file, Click Here or click the hyperlink on the course management page. Ensure that the contents of the course match the framework of this EML and are initially saved as an .xml file locally within your computer. NOTE #1: With the contents of any of these tags, you must use the <![CDATA[]]> wrapper to "escape" any characters that would not translate as a string to the database server. Some examples of what that may look like can be found here NOTE #2: When referencing images within your EML file, ensure that the name of the images are the same as the images that are going to be uploaded to the server

Once your images (.png, .jpg, .gif) and EML file are ready locally, simply click the Create Course button on the left of the page. From the prompt, select the EML file, and the corresponding images. Wait for the contents to completely load. NOTE: some errors may show. Please read them carefully and fix the issues (e.g. number of quizzes don't match number of units, the course name is already taken Once the uploading is complete, you should be able to see you course visible within the course list on the main page. You may view the contents of the course by clicking the Click here to select course button. You will be redirected to the Course Viewer page and can view the units, and check out the quizzes in the Course Quizzes page by click the Course Quizzes navigation button on the top of the web page.

To edit a course, simply click the edit (i.e. pencil icon) button on the course, and upload the modified EML file, and include all of the original and newly added/removed images through the upload process. To remove a course, simply click the delete (i.e. trash can icon) button and the course will be removed, and will not be visible to Learners and anyone else using the site.
